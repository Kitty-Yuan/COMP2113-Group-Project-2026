📖 Project Overview

Dungeon Adventure is a terminal-based RPG game written in C++17 using ncurses library. Players navigate through procedurally generated dungeon mazes, encounter random events and monsters, manage inventory, and complete challenging bosses across multiple difficulty levels.

Key Features

  • 🗺️ Procedurally generated 2D maze dungeons (4 difficulty levels: 9x9 to 20x20)
  • ⚔️ Turn-based combat system with 40% special ability trigger rate
  • 🎮 4 unique monster types with 2 special abilities each
  • 👑 Boss battle with 100 HP and animation sequences
  • 🎪 Side mini-games (Princess chase, Mushroom collection)
  • 💾 User authentication and game progress saving (binary format)
  • 🎨 Colorful ncurses UI with animations and ASCII art
  • 📊 Level-up system, shop, item collection

Technology Stack

  • Language: C++17
  • UI Framework: ncurses (terminal UI library)
  • Build System: Makefile with g++
  • Storage: Binary file format for save data

📦 Data Structures

Player Structure

struct Player
Core player character data
int hp - Current health points (0-100, starts at 100)
int atk - Attack power (affects damage dealt)
int def - Defense power (reduces incoming damage)
int gold - Currency for shopping
int exp - Experience points (triggers levelUp at 100)
int level - Current level (increases with exp)
bool hasKey - Whether player has the dungeon key

Monster Structure

struct Monster
Monster character definition with abilities
string name - Monster name (Ghost, chestnut, Owl, Blob)
string appearance1 - ASCII art frame 1
string appearance2 - ASCII art frame 2 (for animation)
string specialattack1 - Ability 1 name
string specialattack2 - Ability 2 name
string Introduction - Monster description

PlayerStats Structure

struct PlayerStats
UI display version of player stats (includes maxHP)
int hp, maxHP, atk, def, gold, exp, level - Display stats
bool hasKey - Key status indicator

SaveData Structure

struct SaveData (namespace user_save_system)
Complete game state snapshot for persistence
bool valid - Whether save is valid

int hp, atk, def, gold, exp, level - Player stats
bool hasKey, cleared - Game flags

int monsterMin, monsterMax, bossMin, bossMax - Difficulty parameters

Initialization & Setup

Generates new dungeon maze using BFS path carving algorithm. Randomly places: 1 key (K), 1 goal (G), traps (T), chests (C), boss (B), and 30 monsters scattered throughout map.

Player Movement & Interaction

void movePlayer(char m, Player& p, int& px, int& py, char grid[][50], bool visited[][50], int monsterMin, int monsterMax, int bossMin, int bossMax)
Handles player movement in 4 directions (W/A/S/D or arrow keys). Checks collision, updates position, and triggers event based on tile:
  • K - Collect dungeon key
  • T - Trap damage (-15 HP)
  • C - Open chest (+20 Gold with animation)
  • B - Boss battle
  • G - Princess room (win game!)
  • . - Random event
char normalizeMoveKey(int key)
Converts various input formats (W/S/A/D, arrow keys, numpad) into standardized character (W/A/S/D).

Map Display & Pathfinding

void displayMap()
Renders current dungeon map state using ASCII grid. Shows discovered tiles, player position (P), walls (#), and unexplored areas (?).
bool hasPath(int sx, int sy, int ex, int ey, char grid[][50], int SIZE)
BFS algorithm to check if path exists between start and end coordinates through non-wall tiles.
void createPath(int sx, int sy, int ex, int ey, char grid[][50], int SIZE)
BFS path generation. Carves walkable corridor from start to end by clearing wall tiles (used in maze generation).

Level Progression

void levelUp(Player& p)
Triggers when player EXP ≥ 100. Increments level and grants stat bonuses:
  • HP +20
  • ATK +5
  • DEF +3
  • Reset EXP to 0

⚔️ Battle System

Normal Battle

void fight(Player& p, int monsterMin, int monsterMax)
Main turn-based combat loop. Each turn:
  • Display monster animation and player stats
  • Player chooses action: 1) Normal, 2) Strong (-3 HP, 75% hit, 1.3-1.69x damage), 3) Defend (40% success, counter attack)
  • 40% chance to trigger monster special ability (Stage 1: display skill, Stage 2: show damage)
  • Calculate and apply damage both ways
  • Repeat until monster or player reaches 0 HP
Reward on victory: +20 Gold, +50 EXP

Stage 1: Skill Trigger - Displays in bordered frame (x: 8-maxX-8, y: 8-maxY-8):
  • Monster name and ASCII art
  • Ability name in red (e.g., "Ghost USES: Incorporeal Form")
  • Effect descriptions (2-3 lines describing impact)
  • Left sidebar: Player stats panel
  • Right sidebar: 3 battle action buttons (1/2/3)
Waits for Enter keypress before proceeding to Stage 2.
void displayAbilityDamageResult(const Monster& m, int playerDamage, int monsterDamage, bool playerAttackMissed, const Player& p)
Stage 2: Damage Resolution - Shows results in same bordered frame:
  • Title: "=== ATTACK RESULT ==="
  • Player result: "You dealt X damage!" or "Your attack MISSED!"
  • Monster result: "Ghost dealt X damage!"
  • Left sidebar: Updated player stats
  • Right sidebar: Action buttons available

Boss Battle

Special battle vs "DREAD KING" boss (100 HP fixed). Features:
  • Boss has 3 animation frames: idle, charging, slashing
  • Uses standard fight() logic without special abilities
  • Victory triggers game completion (unlock next difficulty)

Renders DREAD KING boss with 3 frame animation (cycles every 8 frames). Shows ASCII art of menacing boss character.

🎪 Event System

Random Events

void event(Player& p, int monsterMin, int monsterMax, int bossMin, int bossMax)
Triggered when stepping on empty floor (.). Rolls random event:
  • 20% - Encounter monster → fight()
  • 15% - Discover shop → shop()
  • 15% - Find mushroom (40% HP, 60% ATK boost)
  • 45% - Safe passage (no event)

Shop System

void shop(Player& p)
Player can purchase 4 mushroom types:
  • Health Mushroom (5 Gold) → +20 HP
  • Strength Mushroom (10 Gold) → +3 ATK
  • Defense Mushroom (10 Gold) → +2 DEF
  • Expensive Mushroom (50 Gold) → +10 HP + 2 ATK
Shows ASCII art mushrooms with color coding (green, red, blue, yellow).

Chest Event

void displayChestEvent(Player& p)
Triggered when stepping on 'C' tile. Displays in bordered frame:
  • Title: "=== TREASURE CHEST ==="
  • Chest ASCII animation (3 frames: closed, closed, open)
  • Gold before: "Before: X Gold"
  • Gold after: "After: X Gold (+20)"
  • 500ms delay between animation frames

Mini-Games

void princessRoomMinigame(Player& p, bool isTrial)
Princess chase side-scroller game:
  • Normal: 30x10 arena, unlimited moves, catch princess to win
  • Tutorial: 10x10 arena, 10-step limit for practice
  • Controls: W/A/S/D to move, princess moves randomly
void tutorialMinigame(Player& p)
Practice version of princess minigame (10x10 map, 10 move limit).

Checks if specified difficulty (1-4) has been completed by player:

  • Returns true if difficulty flag is set
